## 一、项目概述

### 1.1 项目意义

**基于MQTT的车载环境传感器测试系统**旨在构建一套**完整的自动化测试体系**，通过此系统实现传感器数据采集、MQTT传输、自动化测试引擎校验、数据存储及前端展示的闭环流程通过MQTT通信协议，实现对ESP32+DHT11采集数据的可靠传输，集成自动化测试引擎，能够对传感器数据格式、车载工作范围、MQTT通信可靠性等方面进行多维度校验并生成测试报告。

系统测试车载传感器，通过 MQTT 协议上报数据时，在各种情况下（正常、异常、高负载）的表现是否**符合车规要求**。

### 1.2 核心目标

- ✅ **数据采集灵活性**：支持Python模拟生成和真实传感器接入两种模式，实现温湿度数据的灵活采集
- ✅ **MQTT通信可靠性**：基于MQTT协议实现传感器数据的可靠传输，支持QoS级别配置和连接管理
- ✅ **传感器统一管理**：提供统一的传感器配置管理平台，支持多传感器、多模式管理
- ✅ **数据质量验证**：提供数据质量测试功能，验证数据格式、范围、传输可靠性
- ✅ **数据持久化存储**：实现传感器数据的持久化存储，支持历史数据查询和分析
- ✅ **可视化展示**：提供友好的Web界面，实现传感器配置、数据展示、测试管理等功能
- ✅ **扩展性设计**：预留云端集成、性能测试、AI异常检测等扩展能力

### 1.3 系统定位

**系统定位**：基于MQTT协议的车载环境传感器数据采集与管理平台

**核心价值**：
- 📊 **数据采集平台**：提供灵活的传感器数据采集能力
- 🔄 **MQTT通信平台**：实现基于MQTT协议的可靠数据传输
- ⚙️ **配置管理平台**：提供统一的传感器和MQTT配置管理
- ✅ **数据质量验证**：提供数据质量测试和验证功能（辅助功能）
- 💾 **数据存储平台**：实现数据的持久化存储和查询

**系统特点**：
- 以**数据采集与管理**为核心，测试功能为数据质量验证的辅助手段
- 支持模拟数据和真实数据的灵活切换
- 基于MQTT协议实现可靠的数据传输
- 提供完整的传感器配置管理能力
- 支持数据质量测试和验证
- 提供友好的Web可视化界面

## 二、核心模块（高优先级）

### 2.1 传感器数据采集模块

**功能描述**：实现温湿度数据采集，支持Python模拟生成和真实传感器接入两种模式，通过MQTT协议发布数据

**优先级**：🔴 最高（数据来源，无此模块则系统无意义）

#### 前端需求

| 功能项 | 描述 |
|--------|------|
| 模式切换界面 | 支持"Python模拟模式""真实传感器模式"（真实模式默认置灰，提示核心模块完成后启用） |
| 模拟模式配置 | 采集频率（1-5秒）、温度范围（-40℃~85℃）、湿度范围（0%~100%）、MQTT QoS级别（0/1/2） |
| 状态显示 | 运行模式、传感器/模拟数据连接状态、数据采集状态、实时温湿度数据 |
| 日志展示 | 数据发布日志（发布时间、主题、数据内容、数据来源：模拟/真实） |

#### 后端需求

| 功能项 | 描述 |
|--------|------|
| Python模拟脚本 | 生成符合车载场景的温湿度模拟数据，支持配置波动范围、采集频率、异常值模拟 |
| MQTT客户端集成 | 集成Eclipse Paho，实现与EMQ X Broker的连接、断线重连及数据发布 |
| 数据标准化封装 | 将模拟/真实温湿度数据封装为标准JSON格式 |
| 配置参数接口 | 接收前端传入的模式切换指令、配置参数并执行 |
| 状态监控接口 | 实现数据采集状态、模式运行状态的后端监控，供前端查询 |
| 真实传感器接口 | 预留ESP32+DHT11真实传感器对接接口，核心模块完成后可直接迭代开发 |

**标准数据格式**：
```json
{
  "sensorId": "DHT11-001",
  "sensorType": "温湿度",
  "temperature": 25.3,
  "humidity": 48.1,
  "timestamp": "2026-01-06T10:35:22+08:00",
  "dataSource": "simulation"
}
```

#### 接口需求

| 接口类型 | 描述 |
|----------|------|
| MQTT连接接口 | 与EMQ X Broker的TCP连接接口（默认端口1883） |
| 配置参数接口 | HTTP RESTful接口，前端传入模式类型、采集频率、模拟波动范围、QoS级别等参数 |
| 状态查询接口 | HTTP/WebSocket接口，前端查询当前运行模式、连接状态、数据采集状态 |

### 2.2 MQTT通信核心模块

**功能描述**：实现MQTT协议的核心通信能力，包括消息的接收、路由、转发，确保传感器数据从发布端到订阅端的可靠传输

**优先级**：🔴 最高（数据传输通道，核心协议）

#### 前端需求

| 功能项 | 描述 |
|--------|------|
| Broker状态显示 | MQTT Broker运行状态（已启动/未启动） |
| 客户端列表展示 | 当前活跃的MQTT客户端（发布端/订阅端）列表 |
| 消息流转日志 | 查看核心MQTT主题的消息流转日志 |
| 连接配置管理 | MQTT Broker地址、端口、认证信息配置 |

#### 后端需求

| 功能项 | 描述 |
|--------|------|
| EMQ X Broker部署 | 本地运行，支持MQTT 3.1.1/5.0协议 |
| 消息路由功能 | 将发布端的传感器数据转发至订阅端 |
| 消息持久化 | 支持QoS 1/2级别的消息持久化，确保断线后消息不丢失 |
| 会话管理 | 管理客户端会话，支持断线重连后的会话续传 |
| 连接配置管理 | 提供MQTT连接配置的增删改查接口 |

#### 接口需求

| 接口类型 | 描述 |
|----------|------|
| Broker客户端连接接口 | 支持发布端、订阅端的CONNECT/CONNACK报文交互 |
| 消息发布/订阅接口 | 支持PUBLISH/SUBSCRIBE/SUBACK/PUBACK等核心报文交互 |
| 消息状态查询接口 | WebSocket接口，前端查询消息流转日志，后端实时推送 |
| MQTT配置接口 | HTTP RESTful接口，提供MQTT连接配置的增删改查 |

### 2.3 传感器配置管理模块

**功能描述**：提供统一的传感器配置管理平台，支持传感器的增删改查、状态控制、配置参数管理

**优先级**：🔴 高（传感器管理核心功能）

#### 前端需求

| 功能项 | 描述 |
|--------|------|
| 传感器列表 | 展示所有已配置的传感器信息（ID、类型、状态、最后活跃时间、描述） |
| 传感器添加 | 支持添加新传感器配置（传感器ID、类型、描述） |
| 传感器删除 | 支持删除已停止状态的传感器 |
| 状态控制 | 支持启动/停止传感器 |
| 配置管理 | 支持全局传感器配置（数据来源模式、采集频率、温湿度范围、QoS级别） |

#### 后端需求

| 功能项 | 描述 |
|--------|------|
| 传感器CRUD | 实现传感器的增删改查操作 |
| 状态管理 | 实现传感器状态的启动/停止控制 |
| 配置管理 | 实现全局传感器配置的增删改查 |
| 数据验证 | 实现传感器ID格式、参数范围的验证 |
| 状态监控 | 实现传感器状态的实时监控 |

#### 接口需求

| 接口类型 | 描述 |
|----------|------|
| 传感器列表接口 | GET /api/v1/sensors，获取所有传感器信息 |
| 传感器添加接口 | POST /api/v1/sensors，添加新传感器 |
| 传感器删除接口 | DELETE /api/v1/sensors/{sensorId}，删除传感器 |
| 传感器状态接口 | PUT /api/v1/sensors/{sensorId}/status，更新传感器状态 |
| 配置查询接口 | GET /api/v1/sensors/config，获取传感器配置 |
| 配置更新接口 | PUT /api/v1/sensors/config，更新传感器配置 |

### 2.4 数据存储模块

**功能描述**：为系统提供数据持久化支撑，负责存储传感器原始数据、配置信息、测试结果，提供高效的数据读写接口

**优先级**：🔴 高（数据持久化支撑）

#### 前端需求

| 功能项 | 描述 |
|--------|------|
| 历史数据查询 | 支持按sensorId、时间范围查询历史传感器数据 |
| 历史结果查询 | 支持按测试时间、用例ID查询历史测试结果 |
| 数据导出 | 支持原始传感器数据、测试结果的导出（Excel格式） |
| 数据可视化 | 支持生成传感器数据的时序图表（折线图） |
| 存储状态显示 | 显示数据存储占用空间及存储状态 |

#### 后端需求

| 功能项 | 描述 |
|--------|------|
| SQLite数据库构建 | 基于SQLite构建数据库，设计核心表结构 |
| 数据增删改查 | 实现数据的增删改查操作，确保数据写入/查询的高效性 |
| 时间分区存储 | 支持数据按时间分区存储（可选，优化历史数据查询性能） |
| 数据备份功能 | 实现数据备份功能（防止数据丢失） |
| 数据导出功能 | 实现数据导出为Excel格式 |
| 图表生成功能 | 实现数据图表的生成（返回图表URL和数据点） |

**数据库表结构**：

| 表名 | 字段 | 描述 |
|------|------|------|
| sensor_data | dataId, sensorId, temperature, humidity, receiveTime, source | 传感器原始数据表 |
| test_case | caseId, name, type, description, enabled | 测试用例表 |
| test_result | testId, sensorId, caseId, result, testTime, details | 测试结果表 |

#### 接口需求

| 接口类型 | 描述 |
|----------|------|
| 数据写入接口 | 接收传感器原始数据、测试结果，写入数据库（后端内部调用） |
| 数据查询接口 | HTTP RESTful接口，支持前端按sensorId、时间范围、用例ID等多条件查询数据 |
| 数据导出接口 | HTTP接口，提供Excel格式数据文件下载 |
| 图表生成接口 | HTTP接口，生成数据图表并返回图表URL和数据点 |
| 数据备份接口 | HTTP接口，接收前端备份指令，后端执行备份并返回结果 |

## 三、扩展模块（中/低优先级）

### 3.1 数据质量测试模块（中）

**功能描述**：提供数据质量测试功能，验证传感器数据的格式、范围、传输可靠性，确保数据质量符合要求

**优先级**：🟡 中（数据质量验证手段）

#### 前端需求

| 功能项 | 描述 |
|--------|------|
| 测试用例列表 | 展示测试用例列表（用例ID、用例名称、测试类型、状态） |
| 测试操作控制 | 支持测试用例的"批量执行""单条执行""暂停"操作 |
| 实时结果展示 | 实时展示测试结果（通过/失败）及失败原因（如"温度超出范围：90℃＞85℃"） |
| 测试报告管理 | 支持测试报告的生成、预览及导出（Excel/HTML格式） |

#### 后端需求

| 功能项 | 描述 |
|--------|------|
| 测试用例库 | 基于Pytest框架构建测试用例库，覆盖数据格式校验、车载温湿度范围校验、MQTT QoS可靠性校验 |

## 四、技术架构说明

### 4.1 系统架构设计

本项目采用分层架构设计，各层职责清晰：

```
┌─────────────────────────────────────────────────────────────┐
│                      表现层（前端）                            │
│  Vue.js + TypeScript + Tailwind CSS                         │
│  职责：用户交互、数据展示、调用后端API                        │
└─────────────────────────────────────────────────────────────┘
                              ↕ HTTP/WebSocket
┌─────────────────────────────────────────────────────────────┐
│                      应用层（后端）                            │
│  Flask + Python                                              │
│  职责：业务逻辑处理、API接口提供、数据管理                    │
└─────────────────────────────────────────────────────────────┘
                              ↕ MQTT
┌─────────────────────────────────────────────────────────────┐
│                      通信层                                   │
│  EMQ X Broker                                                │
│  职责：MQTT消息路由、持久化、会话管理                         │
└─────────────────────────────────────────────────────────────┘
                              ↕ MQTT
┌─────────────────────────────────────────────────────────────┐
│                      数据采集层                               │
│  Python模拟 / ESP32 + DHT11                                 │
│  职责：传感器数据采集、数据封装、MQTT发布                     │
└─────────────────────────────────────────────────────────────┘
                              ↕
┌─────────────────────────────────────────────────────────────┐
│                      数据存储层                               │
│  SQLite                                                      │
│  职责：数据持久化存储、查询、备份                             │
└─────────────────────────────────────────────────────────────┘
```

| 层次 | 技术选型 | 说明 |
|------|----------|------|
